# DeskLink 로드맵

DeskLink(맥 서버 ↔ 안드로이드 확장 디스플레이)의 향후 개선 계획. 우선순위는 "체감 완성도"를 빠르게 끌어올리는 순서.

## 1순위 — 코어 UX 마무리

- **해상도 변경 시 재적용**: 재연결 시 클라이언트가 새로 선택한 해상도로 가상 디스플레이·인코더를 재구성(리부트). 해상도가 바뀔 때만 재생성하고, 비트레이트만 바뀌면 `updateBitrate`로 처리.
- **맥 Settings 창 구현**: 현재 "Settings..." 버튼이 빈 스텁. 포트, 기본 해상도, 자동 시작, 코덱 등 설정.
- **연결 상태 HUD (태블릿)**: 해상도·FPS·비트레이트·E2E 지연을 화면에 작게 표시. 튜닝/디버깅에 유용.
- **재연결 UX 다듬기**: 상태 메시지 명확화, 수동 재연결 버튼, 미러링 화면의 연결 끊기/설정 오버레이(완료).

## 2순위 — 화질·입력 품질

- **적응형 비트레이트**: 이미 정의된 `BITRATE_UPDATE` 메시지를 활용해 네트워크/CPU 상황에 따라 비트레이트를 자동 조절 → 끊김 감소.
- **HiDPI(레티나) 렌더링**: 가상 디스플레이를 2x 스케일로 만들어 글자·선 선명도 향상.
- **오디오 전송**: ScreenCaptureKit의 시스템 오디오 캡처를 사용해 맥 소리를 태블릿으로 전송(신규 오디오 채널 또는 기존 채널 확장).
- **스타일러스/필압 + 키보드 입력**: 펜 좌표·필압 전송, 태블릿 키보드 → 맥 키 입력 역전송.

## 3순위 — 배포·안정성

- **보안/페어링**: 현재 연결에 인증 없음. 페어링 코드/토큰 추가(특히 Wi-Fi 모드에서 중요).
- **Wi-Fi 모드 다듬기**: mDNS/Bonjour로 서버 자동 검색 → IP 수동 입력 제거.
- **정식 앱 패키징 + 공증(notarization)**: 배포용 .dmg. 단, `CGVirtualDisplay`가 비공개 API라 App Store 배포는 불가.
- **자동 시작 / 메뉴바 다듬기**.

## 추천 진행 순서

해상도 재적용 → 연결/화질 HUD → 적응형 비트레이트 → 오디오 → HiDPI.

## 알려진 제약

- `CGVirtualDisplay`는 비공개 API → macOS 버전 의존성, App Store 배포 불가, 잦은 재생성 시 불안정 가능.
- 맥 앱은 화면 기록·손쉬운 사용 권한을 위해 서명된 .app로 실행해야 함(개발용 자체 서명 인증서 사용 중).
